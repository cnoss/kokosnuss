JavaScript

Basics

WBA1 Workshop Javascript Basics

Where to?

The JavaScript is located in the .js files and is loaded into the HTML using the script tag at the end of your HTML file. This makes sure the script starts running when the rest of your site has finished loading. To load your .js files you write the location into the ref attribute of your script tag. <script ref="example.js"></script>

Variables

Variables in JavaScript are dynamically typed. Instead of a specific data-type you use the var keyword. So each variable changes its specific type depending on their value. After the declaration you can give them a new value at every time. var a = 0;
a = 1;
a = "text";
a = [12, 23, 34]; Aside from primitive types you can assign references to the variables. In that case they don't contain values but simply a redirection to the value of another variable. var b = a;
b[1] = 45; In the example a and b now refer to the same array which contains the numbers [12, 45, 34].

Algorithmic structures

JavaScript uses the same conditionals as java: if, else if, else, switch. And also the same loops: while, for. With those you can write simple algorithms for your page. if (condition) { code }
else if (condition) { code }
else { code }
switch (variable) { case 'value': code }
while (expression) { code }
for () { code } Some algorithms can be written shorter in JavaScript. In conditional assignments one if else condition can be simplified into one line. In the example the value of x is supposed to be "y is 4" if the value of y is 4 and "y is not 4" if it is not. x = (y==4) ? "y is 4" : "y is not 4";

Functions

Functions can be declared using function and the name for your function. The rest works similar to java or other programming languages. As alternative to a declaration you can use a function expression. That way you can use a variable to create a function. function addition (a, b) { total = a + b;
return total; }
var add = function (a, b) { total = a + b;
return total; } You can use these functions like usual functions in other programming languages by using their name and give parameters if necessary. They can be nested and called back. Each nested function can "see" the surrounding functions, but not the other way around. So in the script inner functions are executed first followed by the next bigger functions.

DOM

The DOM, also known as Document Object Model, is a tree structure that resembles your HTML document. The root of the DOM is the document element. By using this element you can gain access to every other element using an element selector or a query selector. The classic method was referring to the wanted element with getElementBy..("..") and then either search for an Id, a ClassName or a TagName. A newer and easier method is to search with querySelector("css-selector") which simply uses a css selector to search and looks the same whether you search for a class name or an id. var a = document.getElementById("first");
var b = document.querySelector("#first"); Both work equally fine but querySelector is the most common. The elements you select this way are nodes in the DOM-tree. Each node has certain attributes which are classList, className, Id, innerHTML, Style, tagName and also sometimes more specific attriutes like href, src, value that aren't available for every node. You can simply save the attribute values or modify them with JavaScript. var c = document.querySelector("#first").className;
document.querySelector(#second).innerHTML = "This is 'second'"; Some attributes have multiple functions, classList for example has .add("className"), .toggle("className"), .remove("className"). To find out what attributes each node has and which functions those attributes have you check the w3schools website.

Events and Listeners

Events are actions that can be detected by JavaScript. Either through properties or listeners. Actions can be caused by user actions like clicks or by the browser for example if it has finished loading the page. When an event is triggered it gets handled by your JavaScript. This can go direktly with simple actions or you call a function with the listener or directly write an anonymous function into the listener. var myButton = document.querySelector("#example");
myButton.onclick = alert("message");
myButton.addEventListener('click', message());
myButton.addEventListener('click', function() { alert("message"); } Whenever an event is triggered an object gets constructed by the browser that contains information about the event. For example the x and y coordinates of a click. These can be used as parameters for your functions. For special functions and how to use them you can check the links at the top.

Git

Tutorial

Hier nochmal schnell ein Tutorial mit den wichtigsten Befehlen, solltet ihr irgendwo nicht weiter wissen mit git. Richtet euren Git-Clienten so ein, dass euer Name da steht, wenn ihr was hochgeladen habt: git config --global user.name "Max Musterheini"
git config --global user.email max@musterheini.com
Pfad wechseln, dahin wo git später das Repo herunterladen soll cd /mnt/x/.../
Um euch das Repo herunterzuladen git clone "URL"
Danach noch einmal den Pfad ändern cd /mnt/x/.../kokosnuss/
Allgemein nachschauen wie es aussieht, was muss gepusht werden, wo wurde was geändert git status
Um in einem eigenem Branch zu arbeiten (am Ende kommt alles in den Master!) git checkout -b 'branchname' (damit wird ein neuer branch erstellt und ihr wechselt direkt in diesen)
git checkout 'branchname' (wechselt in einen anderen Branch)
git branch -d 'branchname' (Branch löschen, falls er nicht mehr gebraucht wird)
Immer bevor ihr was hochladen wollt, checken ob nicht schon wer anderes was geändert habt git pull origin "branchname
Übliche Reihenfolge wenn ihr etwas hochladen wollt: 1. git add -A (fügt alles was ihr gemacht habt hinzu)*
2. git commit -m "Hier reinschreiben was ihr alles gemacht habt"
3. git push origin "branchname"
* alternativ: git add 'filename' (falls ihr nur eine bestimmte File hochladen möchtet)
